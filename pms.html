RASTA POI Management System

Realtà Aumentata e Story-Telling Automatizzato – Sistema di gestione dei Punti di Interesse (POI)

1. Nome del Software

RASTA Poi-Management System

2. Descrizione

Il POI Management System (PMS) è progettato per gestire i Punti di Interesse (POI), permettendo ai contributori di creare, modificare e pubblicare informazioni. Il sistema include funzionalità di front-end e back-office: la parte pubblica mostra i dati inseriti, mentre il back-office offre un'interfaccia per l'inserimento e gestione.

3. Tecnologie Utilizzate

Docker, Azure, Apache Web Server, EMQX, InfluxDB, Telegraf, Grafana, Spring Boot, Spring AI, OpenAI, Apache Mahout, Apache Stanbol (deprecato). Per i dettagli completi, consultare il Deliverable D1.2.

4. Requisiti

  • Windows, Mac OS X, Unix
  • OpenJDK 21 (o Eclipse Adoptium)
  • Apache Maven
  • MySQL (incluso in MAMP per uso locale)

5. Installazione

1. Creare il DB 'rastapms' con:
   username = rastapms
   password = rastapms

2. Eseguire lo script SQL: rasta-pms.sql

3. Compilare:
   mvn clean compile

4. Avviare:
   mvn spring-boot:run

5. Aprire il browser su:
   http://localhost:8080/rastapms/

6. Usage

Repository di codice e installazione:

  • official-rasta-pms
  • rasta-me (REST Services)
  • official-rasta-or6 (istanza OR6)

Il login si effettua tramite l’icona con la freccia in alto a destra nella homepage.

La gestione dei POI include arricchimento semantico con OpenAI e categorizzazione automatica.

Sono presenti 21 descrizioni POI del Parco Nazionale d’Abruzzo.

7. Configurazione

L’architettura è completamente dockerizzata. I dettagli di deployment sono nel Deliverable D2.6.
