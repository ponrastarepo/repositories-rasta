Nome del Software

Implementation of the IoT infrastructure and Dashboards

Descrizione

This section provides the deployment details of the IoT infrastructure that is part of the architecture proposed in Task OR1.6 and OR2.6. It also describes the design and functionality of the associated dashboards used in the implementation.

Tecnologie Utilizzate

Docker, Azure, Apache Web Server, EMQX, InfluxDB, Telegraf, Grafana, Spring Boot, Spring AI, OpenAI, Apache Stanbol, Apache Mahout. For full details see Deliverable D1.2.

Requisiti

  • Docker must be installed on the machine.

Installazione

EMQX

docker pull emqx

docker network create --driver bridge influxdb-telegraf-net

sudo docker volume create --name emqx-data
sudo docker volume create --name emqx-log

sudo docker run -d --name emqx-rasta -e "EMQX_NODE_NAME=emqx@rasta.emqx.com" -p 18083:18083 -p 1883:1883 -v emqx-data:/opt/emqx/data -v emqx-log:/opt/emqx/log --network influxdb-telegraf-net --network-alias rasta.emqx.com emqx:latest

# Per altre istanze (non open source):
sudo docker run -d --name emqx-rasta -e "EMQX_NODE_NAME=emqx@rasta.emqx.com" -v emqx-data:/opt/emqx/data -v emqx-log:/opt/emqx/log --network influxdb-telegraf-net --network-alias rasta.emqx.com emqx:latest

docker exec -it emqx-rasta bash
emqx ctl cluster join emqx@changes.emqx.com

InfluxDB

docker run --name influxdb2 --publish 8086:8086 \
--mount type=volume,source=influxdb2-data,target=/var/lib/influxdb2 \
--mount type=volume,source=influxdb2-config,target=/etc/influxdb2 \
--env DOCKER_INFLUXDB_INIT_MODE=setup \
--env DOCKER_INFLUXDB_INIT_USERNAME=<influx-rasta.user> \
--env DOCKER_INFLUXDB_INIT_PASSWORD=<password> \
--env DOCKER_INFLUXDB_INIT_ORG=rasta \
--env DOCKER_INFLUXDB_INIT_BUCKET=rasta \
--network influxdb-telegraf-net \
--name influxdb influxdb:2

Telegraf

nano <path>/telegraf.conf

docker run -v <path>/telegraf.conf:/etc/telegraf/telegraf.conf:ro \
--network influxdb-telegraf-net --name telegraf --detach telegraf

Contenuto minimo telegraf.conf

servers = ["tcp://rasta.emqx.com:1883"]
username = <mqtt-user>
password = <mqtt_password>

tag_keys = ["clientid"]
fieldpass = ["unit"]

Grafana

docker volume create grafana-storage

docker run -d --name=grafana \
--network influxdb-telegraf-net \
--volume grafana-storage:/var/lib/grafana \
-p 80:3000 \
-e "GF_SERVER_DOMAIN=<server_url>" \
grafana/grafana

Usage

To manage the authorized users on CloudFlare to grant access to the RASTA dashboards:

Each DNS request is first verified by CloudFlare (authorization and authentication). If valid, it is routed to the Azure VM, where an internal basic authorization is also applied. Users were grouped into teams and granted access using one-time access tokens via CloudFlare.

Configurazione

For Docker-based configuration of Telegraf:

nano <path>/telegraf.conf

docker run -v <path>/telegraf.conf:/etc/telegraf/telegraf.conf:ro \
--network influxdb-telegraf-net --name telegraf --detach telegraf

Parametri telegraf.conf

servers = ["tcp://rasta.emqx.com:1883"]
username = <mqtt-user>
password = <mqtt_password>
tag_keys = ["clientid"]
fieldpass = ["unit"]
