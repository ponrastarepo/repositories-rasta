RASTA PMS – POI e Itinerari Parco d'Abruzzo

Modulo per l'aggiunta e la gestione di Punti di Interesse e Itinerari nel Parco Nazionale d'Abruzzo

Nome del Software

Aggiunta di Itinerari e Punti di Interesse nel RASTA PMS
(instanziazione dei moduli sviluppati nel contesto del Parco Nazionale d'Abruzzo)

Descrizione

Il software è stato sviluppato per facilitare l’aggiunta e la gestione delle descrizioni dei Punti di Interesse (POI) all’interno del sistema di gestione RASTA PMS. Le descrizioni reali dei POI del Parco Nazionale d'Abruzzo sono state incorporate come dataset iniziale.

Tecnologie Utilizzate

Le tecnologie e gli strumenti utilizzati includono: Docker, Azure, Apache Web Server per l'infrastruttura, EMQX, InfluxDB, Telegraf e Grafana per la raccolta, memorizzazione e visualizzazione dei dati IoT in tempo reale. I servizi backend sono sviluppati con Spring Boot e Spring AI, con arricchimento semantico e supporto AI forniti da OpenAI, Apache Stanbol e Apache Mahout. Per maggiori dettagli, consultare il Deliverable D1.2.

Requisiti

  • È necessario avere Docker installato sulla macchina.

Installazione

Per l’installazione locale e l’aggiunta di POI e itinerari del Parco Nazionale d’Abruzzo, eseguire lo script pna-data.sql sul database rastapms. È possibile utilizzare un terminale o uno strumento come phpMyAdmin.

Come prerequisito, seguire le istruzioni di installazione ufficiali disponibili all’indirizzo:

https://github.com/gssi/official-rasta-pms/blob/main/Install.md

Comandi per l’ambiente di produzione

Linux/macOS:
docker exec -i mysql-db mysql -u rastapms -prastapms rastapms --default-character-set=utf8mb4 < ./pna-data.sql

Windows:
type .\pna-data.sql | docker exec -i mysql-db mysql -u rastapms -prastapms rastapms --default-character-set=utf8mb4

Utilizzo

Repository con il codice sorgente per l’aggiunta di itinerari e POI:

https://github.com/gssi/official-rasta-or6

Una volta effettuato l’accesso al sistema RASTA PMS, l’utente può visualizzare e gestire gli itinerari e i POI aggiunti. Dal menu laterale, selezionando “Point of Interest”, viene mostrato l’elenco dei POI, basato sulle informazioni tratte da:

https://www.parcoabruzzo.it/page.php?id=251

Configurazione

L’intera architettura è modulare e si basa su Docker. La procedura di distribuzione è descritta nel Deliverable D2.6.
